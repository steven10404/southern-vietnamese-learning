# 三輪審查筆記

更新日期：2026-04-29

## 第一輪：南越越南語老師視角

目前優點：

- 生活情境選得對：點餐、交通、市場、問路、外送、旅館、工作都很實用。
- 南越發音方向正確：hỏi/ngã 合流、d/gi/v 接近 y、tr/ch 與 s/x 合流、尾音變軟都已納入。
- 詞句比單字更適合初學者，這點很適合口說導向學習。

一定要補的內容：

- 稱謂系統：em、anh、chị、cô、chú、con、mình。
- 禮貌請求：Cho em、Cho em xin、giúp em。
- 問句：không、chưa、hả。
- 時間與狀態：đang、rồi、chưa、mới、sắp。
- 語尾詞：nha、ha、hả、hen、thôi。
- 量詞：cái、con、ly、tô、dĩa、phần、ký。
- 南越口語核心詞：vô、quẹo、xài、mắc、chút xíu、bữa nào。

已處理：

- 已新增 `grammar` 資料庫區塊。
- 已新增「一定要會的南越口語文法」頁面區塊。
- 已讓 RAG 可以檢索文法內容。

## 第二輪：資深技術員視角

目前優點：

- `data/content.json` 把內容和程式分離，已經有資料庫雛形。
- RAG 區塊目前雖然不是 LLM，但資料流正確：問題 → 檢索 → 引用 → 回答。
- 沒有把 API key 放在前端，這點是對的。

主要風險：

- 現在是關鍵字檢索，不是語意 embedding；相似問題仍可能抓偏。
- 回答仍是模板整理，不是真 LLM 回答。
- 資料庫沒有 chunk id、source id、review status 的完整關聯表。

已處理：

- RAG 已納入 `grammar`。
- 針對口音/文法/詞句/對話問題加了不同權重。
- RAG 回答已加 HTML escape，避免使用者輸入直接插入頁面。

下一步：

- 新增 `POST /api/rag`，由後端處理 embeddings 和 LLM。
- 建立 `rag_chunks`：每筆 chunk 保留 `type`、`source`、`review_status`、`text`。
- 使用 SQLite 或本機向量檔保存 embedding。

## 第三輪：產品/客戶經理視角

目前優點：

- 視覺很有記憶點，第一眼就知道是南越學習工具。
- 單頁長頁比早期多頁更適合目前內容量。
- 導覽順序大致合理：發音 → 對話 → 文法 → 資料庫 → RAG → 音檔 → 來源。

產品缺口：

- 缺少「學習進度」概念：今天學了什麼、下次複習什麼。
- 缺少「任務型課程」：例如 7 天南越口音入門。
- RAG 區塊目前仍偏工程展示，之後應改成「問老師」語氣。
- 來源與校對狀態應逐步變成信任機制，而不是只有備註。

建議路線：

1. 保留目前頁面視覺。
2. 加入 7 天入門路線。
3. 詞句卡支援「已會 / 還不熟 / 想真人校對」。
4. RAG 區塊命名可逐步改成「問南越老師」。
5. 真人音檔補上後，把音檔狀態顯示在詞卡上。
